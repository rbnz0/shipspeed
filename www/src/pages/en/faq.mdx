Here are some commonly asked questions about ShipSpeed.

## What's next? How do I make an app with this?

We try to keep this project as simple as possible, so you can start with just the scaffolding we set up for you, and add additional things later when they become necessary.

If you are not familiar with the different technologies used in this project, please refer to the respective docs. If you still are in the wind, please join our [Discussions](https://github.com/rbnog/shipspeed/discussions) and ask for help.

- [Next.js](https://nextjs.org/)
- [Better Auth](https://www.better-auth.com)
- [Prisma](https://prisma.io)
- [Tailwind CSS](https://tailwindcss.com)
- [tRPC](https://trpc.io)
- [Drizzle](https://orm.drizzle.team/docs/overview)

## How do I keep my app up to date?

ShipSpeed is a scaffolding tool, not a framework. This means that once you initialize an app, it's yours. There is no postinstall CLI tool to help you stay up to date. If you want to keep track of any improvements we make to the template, you could [enable notifications for releases](https://docs.github.com/en/account-and-profile/managing-subscriptions-and-notifications-on-github/setting-up-notifications/configuring-notifications#configuring-your-watch-settings-for-an-individual-repository) on our repository.

## What learning resources are currently available?

The community recommends that you just start using the stack and learn along the way by building with it.

If you are considering ShipSpeed, chances are you might have already used some of the parts of the stack. So why not just dive in head first and learn the other parts while you build something?

## Why are there `.js` files in the project?

As per our [typesafety principle](/en/introduction#typesafety-isnt-optional), we treat typesafety as a first class citizen. Unfortunately, not all frameworks and plugins support TypeScript which means some of the configuration files have to be `.js` files.

We try to emphasize that these files are JavaScript for a reason, by explicitly declaring each file's type (`cjs` or `mjs`) depending on what's supported by the library it is used by. Also, all the `js` files in this project are still typechecked using a checkJs option in the compiler (tsconfig).

## I'm struggling to add i18n to my app. Is there any reference I can use?

We have decided against including i18n by default in `create-shipspeed` because it's a very opinionated topic and there are many ways to implement it.

## Should I use `/app` from Next.js 13 or the more mature `/pages` paradigm?

You have the option when scaffolding an app using ShipSpeed to opt into the `/app` directory structure. As of the time of writing, this feature is generally considered mature enough to be used in production.

Nevertheless, if you strongly prefer to use the old `/pages` paradigm, that's still an option. Porting over your existing router can be a monumental effort, so do not feel undue pressure to do so unnecessarily.
